## ğŸ“ Grading Calibration Notes

### When Students Are Between Excellent & Poor

**Common In-Between Patterns:**

**B-Level Work (18-22 points):**
- Makes one clear change (follows rules)
- Some trade-off analysis but not deep
- Character mostly preserved
- Decent documentation
- Basic reflection without deep insights

**What Pushes B to A:**
- Exceptional insight in trade-off analysis
- Transfer learning evident
- Creative solution within constraints
- Professional-quality documentation
- Sophisticated meta-cognitive reflection

**What Drops C to D:**
- Multiple changes (breaking core rule)
- Complete character removal
- No testing or documentation
- Generic or missing reflections
- Doesn't address the problem identified

### Red Flags Requiring Discussion

**Possible Misunderstanding:**
- Claims "no trade-offs exist"
- "It's perfect now" with no remaining flaws
- Defensive about maintaining any friction
- Missing the constraint's purpose

**Response:** Meet with student to discuss design thinking and the value of constraints.

**Possible Academic Integrity Issue:**
- Work quality dramatically exceeds other submissions
- Can't explain their own process
- No iteration visible
- Copied from external source

**Response:** Request explanation of process, check for plagiarism, follow institution policy.

## ğŸ”„ Using Examples for Different Purposes

### Week 1: Setting Expectations
Show excellent example (with student permission):
> "This is what A-level work looks like. Notice the trade-off analysis and meta-learning. You'll get there!"

### Week 3: Assignment Introduction
Share both examples (names removed):
> "Here are two approaches to this assignment. What makes one excellent and the other miss the point?"

### Week 4: Peer Review Training
Have students practice grading these:
> "Grade these two submissions using the rubric. Then we'll compare notes and discuss."

### Week 8: Portfolio Reflection
Show progression:
> "Look at your Week 3 work vs. these examples. How has your understanding grown?"

## âš ï¸ Important Notes

### Privacy Considerations
- Always get student permission before sharing work
- Remove names when using for general examples
- Don't identify "poor" examples as coming from specific students
- Focus on learning, not shaming

### Maintaining Currency
- Update examples each semester based on actual submissions
- Add examples for new assignment types
- Remove outdated examples (if tools/requirements change)
- Keep excellent examples that demonstrate timeless principles

### Avoiding Grade Inflation
- Don't lower standards to match average work
- Maintain distinction between good and excellent
- Explain WHY excellent is excellent (not arbitrary)
- Show improvement pathways from any starting point
